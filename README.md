# 🎤 豐原向陽店 KTV 分帳計算機

一個純前端的單頁工具，依「人數、日期類型、進場時間、歡唱時數」自動算出**每人應付金額**，並支援**企業合作折扣**。資料來自豐原向陽店消費方式計價表。

## 功能

- 下拉選擇人數（最多 30 人），自動帶入對應包廂（可手動更改）
- 四種日期類型：平日（一~四）、星期五、星期六、星期日，各自的時段折數
- 依**進場時間**判定時段；**跨時段按分鐘計算價差**
- 「買二送 N」自動折減計費時數（每 2+N 小時只收 2 小時；鐘點計價則全收）
- 依進場時段**自動帶入歡唱時數**（買二送 N＝共 2+N 小時）；鐘點計價時段則可自選時數
- 顯示**離場時間**與**歡唱總時數**
- 企業合作折扣（目前含「台中美光 85 折」），只套用於包廂費、且在時段折扣之後
- 即時顯示總計與每人金額、完整費用明細
- **⚙️ 設定面板**：右上角齒輪按鈕可直接編輯所有價格與折扣規則（見下方說明）

## 計費規則

```
包廂費 = Σ(每分鐘包廂原價 × 該分鐘所在時段折數) × 企業折扣   （僅計費時段，免費時段不計）
清潔費 = 清潔原價 × 清潔折數   （08:00~16:59 五折、其餘原價、星期日全天原價；企業折扣不打）
服務費 = (包廂原價 × 計費時數 + 清潔原價) × 10%   （以折扣前原價計算；贈送的免費時數不計、企業折扣不打）
總計   = 包廂費 + 清潔費 + 服務費
每人   = 總計 ÷ 實際人數
```

> 跨時段、現場活動及實際收費以門市現場為準，本工具僅供估算參考。

## 發布到 GitHub Pages

1. 建立一個新的 GitHub repository，把 `index.html` 推上去。
2. 進入 repo 的 **Settings → Pages**。
3. **Source** 選 `Deploy from a branch`，分支選 `main`、資料夾選 `/ (root)`，儲存。
4. 稍候即可透過 `https://<你的帳號>.github.io/<repo 名稱>/` 開啟使用。

本工具為 `index.html` + `config.json`，無任何相依套件。直接用瀏覽器開啟本機 `index.html` 也可使用（此時讀不到 `config.json`，會自動退回內建預設價目，功能不受影響）。

## 檔案說明

| 檔案 | 用途 |
|---|---|
| `index.html` | 工具本體（介面、計算邏輯、設定面板）。內含一份預設價目作為後備。 |
| `config.json` | **共用價目表設定來源**，工具啟動時載入。要讓所有人看到的價格，改這個檔。 |
| `pricing.jpg` | 原始計價表圖片（參考用）。 |

## ⚙️ 設定面板（修改價目表）

點工具右上角的齒輪按鈕，即可在網頁上直接編輯：

- **包廂價格**：各包廂的包廂費/時、清潔費、人數範圍
- **企業合作優惠**：新增／刪除公司、設定折數（百分比）
- **時段折扣**：每個時段的時間區間、是否鐘點計價、買二送幾小時、折數

### 兩種儲存層級（重要）

1. **按「儲存」** → 修改只存到**你這台電腦的瀏覽器**（localStorage），方便臨時試算；其他人看到的仍是舊價。
2. **要讓所有人都更新** → 需把設定同步回 `config.json`。設定面板內已附**完整操作 SOP** 與「🔗 開啟 GitHub 編輯頁」按鈕：

   1. 在面板按「📋 複製設定碼」
   2. 開啟 `https://github.com/<OWNER>/<REPO>/edit/main/config.json`
   3. 全選舊內容刪除、貼上設定碼
   4. Commit changes
   5. 等 1~2 分鐘，GitHub Pages 自動更新，所有人重新整理即見新價

> 載入優先序：**本機修改（localStorage） > config.json > index.html 內建預設**。
> 在面板按「重設為預設」會清除本機修改，回到 `config.json` / 官方價目。
